## Ordered Implementation Plan

### Phase 1 — Database migration and migration runner

1. Add `migrations/0009_campaign_memberships_role_unification.sql`:
   - Rebuild `campaign_memberships` as constrained table (`role IN ('member','gm')`).
   - Preserve uniqueness on `(user_id, campaign_slug)` and supporting indexes.
   - Copy valid existing membership rows.
   - Upsert `campaign_gm_assignments` rows as `role='gm'`.
   - Conflict policy: overlapping rows resolve to `gm`.
   - Preserve existing membership `id` when present; generate deterministic id for GM-only backfill rows.
   - Keep `campaign_gm_assignments` table intact for burn-in.

2. Add `migrations/0010_drop_campaign_gm_assignments.sql`:
   - Drop legacy table only.
   - File exists for later release; no active execution in this session.

3. Update `scripts/db-migrate-auth-plan.mjs`:
   - Include `0009` in `orderedMigrations` after `0008`.
   - Do **not** include `0010` yet.
   - Add preflight conflict detection for invalid `campaign_memberships.role` values.
   - Include role-quality metric/reporting in dry-run output.
