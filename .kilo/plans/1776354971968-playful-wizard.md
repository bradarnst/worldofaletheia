## What Is Next

Recommended next initiative: **Architecture/design session for site-wide search**.

### Search Architecture Session Goals

1. Define the canonical user journeys:
   - global site search
   - scoped collection search
   - campaign-aware protected-result behavior
2. Lock API contract for `/api/search`:
   - request params (`q`, `collection`, filters, pagination)
   - response envelope, facets, and error states
3. Define ranking model and relevance phases:
   - Phase A: metadata/title/summary ranking
   - Phase B: weighted fields + typo tolerance strategy
   - Phase C: optional FTS path (only if needed)
4. Define authz/visibility guarantees:
   - deny-by-default for protected content when auth context is missing/ambiguous
   - explicit behavior for mixed public/protected result sets
5. Define observability and quality gates:
   - latency SLO, result quality checks, stale-index detection, failure handling

### Search Session Deliverables

1. One architecture memo (or ADR if decision scope warrants it).
2. A phased implementation plan with acceptance criteria per phase.
3. A concise test plan covering relevance, authz filtering, and failure modes.
