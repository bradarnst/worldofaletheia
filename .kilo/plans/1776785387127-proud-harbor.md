# Plan: Remove Domain Tab Bars + Reference Overview Page

**Date:** 2026-04-23  
**Status:** Ready for implementation

---

## Context

Two UX issues:

1. **Tab bars are redundant.** The global nav already provides dropdown access to every section. The per-domain section subnav tab bars (World of Aletheia, Using Aletheia, Campaigns) duplicate this, and `UsingAletheiaNav` now has *two* rows (the original links row plus the game systems row added in the previous sprint). All of them can go.

2. **Reference Overview page is redundant.** `/references` just re-lists Calendar, Timeline, and Maps — information already in the global nav dropdown. The page can be removed, its tab bar entry removed, and the `ReferenceNav` reduced to only the three live surfaces.

---

## Scope — Exact Changes

### 1. Remove `WorldAletheiaNav` from both WorldAletheia layouts

**Files:**
- `src/layouts/WorldAletheiaLayout.astro` — remove `import WorldAletheiaNav` + remove `<WorldAletheiaNav />` element
- `src/layouts/WorldAletheiaContentLayout.astro` — same

**Result:** No tab bar on any Canon collection page or article page.

---

### 2. Remove `UsingAletheiaNav` from both Using Aletheia layouts

**Files:**
- `src/layouts/UsingAletheiaLayout.astro` — remove `import UsingAletheiaNav` + remove `<UsingAletheiaNav />` element
- `src/layouts/UsingAletheiaContentLayout.astro` — same

This removes both the "Systems / Meta" row *and* the "Game systems / GURPS" sub-row from every Using Aletheia page. The game systems card section on `/systems/index.astro` (added in the previous sprint) remains — it's in the page body, not a nav bar.

**Result:** No tab bar on any Using Aletheia page.

---

### 3. Remove `CampaignsNav` from both Campaigns layouts

**Files:**
- `src/layouts/CampaignsLayout.astro` — remove `import CampaignsNav` + remove `<CampaignsNav />` element
- `src/layouts/CampaignsContentLayout.astro` — same

**Note:** The `CampaignsNav` login link currently points to `/login?next=<current path>`. That functionality is **not** critical to preserve in a nav bar — login is available from the global nav. If needed, a future authenticated-state indicator can live in the global nav.

**Result:** No tab bar on any Campaigns page.

---

### 4. Remove the Reference Overview page and simplify `ReferenceNav`

#### 4a. Delete `src/pages/references/index.astro`

The page serves no purpose once the nav dropdown already lists the three surfaces. **The file should be deleted.**

#### 4b. Update `src/components/ReferenceNav.astro`

Remove the `Overview` entry from the links array. The nav will only contain:
- Calendar → `/references/calendar`
- Timeline → `/references/timeline`
- Maps → `/references/maps`

Also simplify the `isActive` guard — the special-case `href === '/references'` exact-match logic was only needed because Overview was a prefix match issue. With Overview gone, the standard `startsWith` check works uniformly for all three remaining entries.

#### 4c. Update `src/pages/sitemap.xml.ts`

Remove `'/references'` from the static paths array (the three sub-paths remain).

#### 4d. Update `src/pages/index.astro`

The home page references card for "Reference" currently links to `href: '/references'`. This should be updated to redirect to the first sub-surface, e.g. `href: '/references/calendar'`, or updated to `/references/calendar` as the canonical landing point. Alternatively the description can be left as-is and just the href changed.

#### 4e. Consider `ReferenceLayout.astro`

After Overview is gone, `ReferenceLayout` will still render the `<ReferenceHeader>` for each sub-page (Calendar, Timeline, Maps). This is fine — no change needed to the layout itself.

---

### 5. Clean up orphaned nav component files (optional, low priority)

After layouts stop importing them, these components become dead code:
- `src/components/WorldAletheiaNav.astro`
- `src/components/UsingAletheiaNav.astro`
- `src/components/CampaignsNav.astro`

Recommend **deleting all three** as part of this PR to avoid confusion. `SectionSubnav.astro` itself is still used by `ReferenceNav.astro`... actually wait — `ReferenceNav` does *not* use `SectionSubnav`; it has its own bespoke markup. That means `SectionSubnav.astro` will have **no remaining consumers**. It too can be deleted.

**Deletion list:**
- `src/components/WorldAletheiaNav.astro`
- `src/components/UsingAletheiaNav.astro`
- `src/components/CampaignsNav.astro`
- `src/components/site/SectionSubnav.astro`
